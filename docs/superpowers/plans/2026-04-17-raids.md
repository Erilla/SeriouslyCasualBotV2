# Raids Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the Raids domain: Raider.io service, roster sync with grace period, identity map, auto-link suggestions, missing user alerts, message refresh, overlord management, signup alerts, and weekly M+ / Great Vault reports.

**Architecture:** Raider.io service wraps the external API with native fetch. Roster sync runs every 10 minutes via the scheduler, compares API data against the raiders DB table, applies a 24-hour grace period for missing raiders, auto-links via identity map, and suggests matches based on Discord member names. Signup alerts and weekly reports are cron-scheduled.

**Tech Stack:** TypeScript 6.x, Discord.js 14, better-sqlite3, node-cron, native fetch, vitest

### Task 3: Roster Sync Core Logic

**Files:**
- Create: `src/functions/raids/syncRaiders.ts`
- Create: `tests/unit/syncRaiders.test.ts`

- [ ] **Step 1: Write failing tests**

Test the sync logic with mocked DB and API:
- New raider added to DB when in API but not in DB
- Raider gets `missing_since` set when in DB but not in API
- Raider `missing_since` cleared when they return to API
- Raider flagged after 24h grace period
- Ignored characters excluded from sync
- Discord user auto-populated from identity map for new raiders
- Class fetched from Raider.io character profile for new raiders

- [ ] **Step 2: Implement syncRaiders**

`syncRaiders(client)`:
1. Fetch roster from `raiderio.getGuildRoster()`
2. Get all raiders from DB, all ignored characters from DB
3. Filter API roster: exclude ignored characters
4. For each DB raider not in API roster: set `missing_since` if null, skip if < 24h, flag if >= 24h
5. For each DB raider back in API: clear `missing_since`
6. For each API raider not in DB: check `raider_identity_map` for discord_user_id, insert into raiders
7. Post sync summary to bot-logs channel
8. All DB operations in a transaction

- [ ] **Step 3: Run tests, commit**

```bash
git commit -m "feat: add roster sync with grace period and identity map auto-linking"
```
